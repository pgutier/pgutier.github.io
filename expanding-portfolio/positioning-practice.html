How to compose elements on a page

Below are some examples of various useful position settings for composing elements on pages. Open the code and have a look at how it is done.

A <div> tag is useful for making a generic container that can be styled and composed on a page. Think of it as a box that you can put other elements inside and move around. It has no semantic value.

Whenever possible one should use actual semantic html tags (tags that are representative of the meaning they envelope) for content instead of a div tag.

Read about <article> and <aside> tags. This is a good reference for tag descriptions and proper use. https://developer.mozilla.org/en-US/docs/Web/HTML/Element/article

Search engines will better catalog your website if the content is semantically tagged.

You will also see a <nav> tag on this page. This is the tag used for navigation. I have provided a useful hover effect, which expands the size of the box revealing more informaiton.

This div is not styled, also known as static positioning. It goes with the flow of all the other content. It sits where it is written in the code.

Fixed positioning is a type of absolute positioning that requires the position property to have a value of fixed. It stays in place when the user scrolls down the page.

Absolute positioning takes something out of the normal document flow and it no longer affects the position of other elements on the page. The other elements on the page ignore it. The box offset properties (top, bottom, left, and right) specify where the element should appear in relation to its container.
